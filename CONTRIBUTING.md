# Contributing to AEGIS OMNI-XEON

Contributions are welcome for non-commercial purposes. By contributing, you agree that your contributions will be licensed under the same dual-license terms as the project.

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test thoroughly on a Linux system
5. Submit a pull request with a clear description

## Guidelines

- Keep changes focused and minimal
- Follow existing code style (Python, PEP 8)
- Test on Ubuntu/Debian before submitting
- Do not commit secrets, credentials, or API keys
- Do not add dependencies without discussion

## Reporting Issues

Open a GitHub issue with:
- Clear description of the problem
- Steps to reproduce
- Expected vs. actual behavior
- System info (OS, Python version)

## Code of Conduct

Be respectful, constructive, and professional in all interactions.
